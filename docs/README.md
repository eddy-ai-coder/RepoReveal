## Documentation Structure

Each repository has its own directory with the following structure:

```
docs/{owner}/{repo-name}/
├── repository-overview.md    # Project overview and key information
├── architecture.md            # Architecture diagrams and design
├── technical-details.md       # Implementation details and APIs
└── indepth-guide.md          # Usage guides and best practices
```

## How to Navigate

1. **Browse by repository**: Find the repository you're interested in
2. **Start with overview**: Read `repository-overview.md` first
3. **Understand architecture**: Review `architecture.md` for system design
4. **Deep dive**: Read `technical-details.md` for implementation specifics
5. **Learn by doing**: Follow guides in `indepth-guide.md`
